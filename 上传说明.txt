# 1. 初始化本地 Git 仓库（会在文件夹中创建 .git 目录）
git init

# 2. 添加远程仓库地址（将 URL 替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/仓库名.git

# 3. 添加所有文件到暂存区（“.” 代表当前目录所有文件）
git add .

# 4. 提交文件到本地仓库（-m 后面是本次提交的说明）
git commit -m "首次上传项目文件"

# 5. 将本地分支重命名为 main（与 GitHub 默认分支名一致）
git branch -M main

# 6. 拉取远程仓库内容，避免推送冲突
#    --rebase 参数会让提交历史更干净
git pull --rebase origin main

# 7. 推送本地代码到 GitHub 远程仓库
git push -u origin main

如果你使用 HTTPS 地址，推送时会要求验证：

用户名：输入你的 GitHub 邮箱或用户名。

密码：注意！这里不是 GitHub 登录密码，而是需要输入个人访问令牌（Personal Access Token）。

生成令牌：GitHub 头像 → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token，勾选 repo 权限，生成后复制令牌。

推送时，在密码框粘贴这个令牌即可。

4. 后续更新
以后修改了文件，只需重复以下三步即可同步到 GitHub：
git add .
git commit -m "更新说明"
git push
